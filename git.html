Panduan Lengkap Deploy Website ke GitHub

Apa itu Git dan GitHub?

Git adalah sistem kontrol versi yang memungkinkan Anda melacak perubahan pada kode sumber. GitHub adalah platform hosting berbasis web yang menggunakan Git untuk manajemen versi kode. Dengan menggunakan keduanya, Anda dapat dengan mudah menyimpan, mengelola, dan berbagi kode website Anda.

Persiapan Sebelum Deploy

  • ✓ Install Git di komputer Anda (download dari git-scm.com)
  • ✓ Buat akun GitHub di github.com
  • ✓ Siapkan folder website yang akan di-deploy
  • ✓ Konfigurasi Git dengan email dan username GitHub Anda

Konfigurasi Git

Setelah instalasi Git, set email dan username:

git config --global user.name "username_github_anda" git config --global user.email "email_github_anda"

Cara Membuat Repository Baru

Ada dua cara untuk memulai: membuat repository baru atau menggunakan repository yang sudah ada.

1. Membuat Repository Baru

Jalankan perintah berikut di terminal:

echo "# NamaProject" >> README.md git init git add README.md git commit -m "first commit" git branch -M main git remote add origin https://github.com/username/NamaProject.git git push -u origin main

2. Push Repository yang Sudah Ada

Jika Anda sudah memiliki repository lokal:

git remote add origin https://github.com/username/NamaProject.git git branch -M main git push -u origin main

Penjelasan Perintah Git

Perintah Dasar

  • git init - Inisialisasi repository Git baru
  • git add - Menambahkan file ke staging area
  • git commit - Menyimpan perubahan ke repository
  • git push - Mengirim perubahan ke repository remote
  • git pull - Mengambil dan menggabungkan perubahan dari repository remote

Contoh Penggunaan Lengkap

# Menambahkan semua file git add . # Commit dengan pesan git commit -m "Update website design" # Push ke branch main git push origin main

Tips dan Troubleshooting

Tips Penting

  • ✓ Selalu buat .gitignore untuk mengabaikan file yang tidak perlu di-push
  • ✓ Gunakan branch terpisah untuk fitur baru
  • ✓ Tulis commit message yang jelas dan deskriptif
  • ✓ Lakukan pull sebelum push untuk menghindari konflik
  • ✓ Backup repository lokal secara berkala

Troubleshooting Umum

# Error: Permission denied git remote set-url origin https://username:token@github.com/username/repo.git # Error: Failed to push git pull origin main --rebase git push origin main # Error: Untracked files git add . git stash git pull git stash pop

Deploy Website dengan GitHub Pages

GitHub Pages memungkinkan Anda untuk meng-host website statis langsung dari repository GitHub Anda.

Langkah-langkah Aktivasi GitHub Pages

  1. 1. Buka repository di GitHub
  2. 2. Klik tab Settings
  3. 3. Scroll ke bagian GitHub Pages
  4. 4. Pilih branch yang akan di-deploy (biasanya main)
  5. 5. Simpan, dan tunggu beberapa menit hingga website Anda live

Website Anda akan tersedia di: https://username.github.io/nama-repository
